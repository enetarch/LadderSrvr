Method : Create

Syntax | Description | Parameters | Values | Returns | Remarks | Example | See Also |

Syntax

Public Create ( Folder, szName, szDescription, nClass) : Boolean

Public Create_Folder ( szName, szDescription, nClass) : Folder
Public Create_Item ( szName, szDescription, nClass) : Item
Private Create_Item2 ( szName, szDescription, nClass, szTableName) : Item
Public Create_Reference ( szName, szDescription, nClass, Folder) : Reference

Description

Create initalizes a folder object to the instance data passed in through the parameters. The object will not be stored until STORE is called. That is when the ID initialized.

Create Folder, Item, and Reference, add a new instances of Folder, Data Item, or Reference to the current folder. These instances will not be saved to Ladder's store until STORE is called.

Parameters

[ Folder ] - the parent link, unless this instance will be a root folder.

[ szName ] - The name of the folder to be created.

[ szDescription ] - The description of the folders contents

[ nClass ] - The class of the folder, item or reference.

[ szTableName ] - This SQL table used by this Data Item.

Returns

The container class returns an instance of the Folder, Data Item or Reference used to manage the data.

Remarks

The process of storing data in Ladder starts with creating an instances and storing in. These are two seperate commands - Create and Store. This is done on purpose to avoid unnecessary transactions between the library and the storage engine.
