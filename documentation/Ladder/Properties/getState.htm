Method : getState

Syntax | Description | Parameters | Returns | Remarks | Example | See Also |

Syntax

Public getState ( ) : Array

Description

Each Link Instance's state is transfered through get and set state. This allows objects to overload each other, as new classes consume older classes.

Parameters

Array - contains the object that will take the information or state of the instance and save it to the database.

Returns

[ Array ] - a key value pair of fields and values stored to the database.

Remarks

While get State exposes the objects inner data, it recommended that you don't change these values, and simply pass them to the object being initialized. Changing the state values could overwrite existing data, or perminently damage the tree.

If it is needed, a authentication key or encryption could be employeed to safely pass the data to initialize super objects.

Example

A good example of where get state is used is in Tree. Tree is the RPC interface through a web server that allows the Ladder Client to send, retrieve, and manipulate objects in the Ladder Tree.

The logic here could be extended to validate the data for class type constraints, base type constraints, and parental constraints. (For example, each class has to be a parent of the Root Classes Folder.) 

case "Folder:Create_Folder" :
{
	$objFolder = $gblLadder->getItem ($szParams[0]);
	// Function Create_Folder ($szName, $szDescription, $nClass, $thsObject=null)
	$thsFldr = $objFolder->Create_Folder ($szParams[1], $szParams[2], $szParams[3]);		
	$rtn = $thsFldr->getState();
} break;
