Description

Link Properties, pulls specific information from the Link Table entry and makes it publicly available. Such information as the Created, Modified, and Last Accessed by Date, User and IP Address.

Remarks

This was done to avoid allowing users access to the Link Object. By making this object publicly accessable, the methods used to maintain the references between links .. ie their heirarchy is hidden. In this fashion, it can be updated, changed, expanded, or contracted to meet the systems needs.

XML Schema

<Touched By>
<Date> [ccyy-mm-dd hh:mm:ss] </Date>
<UID> [szString] </UID>
<IP> [x00.x00.x00.x00] </IP>
</Touched By>

<Properties>
<Created> <Touched By> .. </Touched By> </Created>
<Modified> <Touched By> .. </Touched By> </Created>
<Accessed> <Touched By> .. </Touched By> </Created>
</Properties>

Usage Concept

Link properties are the equivelent of DOS File properties. They say who did what when. Ladder was setup to initially mimic the DOS file structure.
