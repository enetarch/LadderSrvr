Description

Ladder is a data management system that uses a heirarchical structure to manage the relationships of information. Ladder works with many existing storage mediums .. from Microsoft Access to Microsoft SQL. The relationships are stored in one table as a tree, while the instance data of each class is stored in several other tables.

The tree is used to store the relationships of the information. As information is gathered and stored, it's relationship to other informaiton can be managed by storing the relationships in a folder. Instances of data can be shared amongst folders simply by creating a link in the corresponding folders to the instance data.

The instance data is stored in seperate tables. The table name is located by reviewing the class information for the instance. For example, if the informaiton stored in the instance pertained to telephoen numbers, then your objects that manage telephone numbers might use a table named "Telephone". They however don't have to, and thus you're free to choose a table name, so long as it doesn't exist with other existing tables.

The Template class manages what infomraiton can be stored in a folder. This can stop unnecessary instances of data from being added to a folder. Or, it can designate how many instances of a particular instance can go into a folder.

Usage Concept

Ladder as a information management system has many uses. One use is as a Daytimer. In a daytimer, information concerning the customer, their projects, the appointments, and notes are all tied together so that they are easily retrievable through many different types of sorts. You might be looking for information related to a meeting, thus knowing the date of the meeting, you can review all the notes taken on that day. Those same notes might be relevant to a project you are working on for the client. Thus the project would have a link to these notes.

Ladder accomplishes this feat by removing the traditional method of referencing information and placing all the references in a ladder tree. At this point, the tree contains folders and sub folders. A Project folder could contain a link to the notes taken, todo items that were derived from that meeting, appointments for the next meeting, and so on.
